# Octane + Swoole Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace PHP-FPM with Laravel Octane + Swoole to keep the app in memory and serve requests at maximum speed.

**Architecture:** Docker container switches from `php:8.3-fpm-alpine` to `php:8.3-cli-alpine` with the Swoole PECL extension. Nginx stays in front as a reverse proxy (static files + proxy to Octane on port 8000). Supervisor manages Octane, Nginx, and Reverb as separate processes. Two static array caches in API resources are fixed to prevent memory leaks.

**Tech Stack:** Laravel 12, Laravel Octane, Swoole, Nginx (reverse proxy), Supervisor, Docker, PostgreSQL 16, Redis 7

**Design doc:** `docs/plans/2026-03-05-octane-swoole-design.md`

### Task 2: Fix Memory Leak in ProductResource

The static `$wishlistProductIds` array persists across requests under Octane, causing stale data and unbounded memory growth. Add a `flushWishlistCache()` method so it can be reset per-request.

**Files:**
- Modify: `app/Http/Resources/ProductResource.php:60-84`
- Test: `tests/Unit/Resources/ProductResourceOctaneTest.php`

**Step 1: Write the failing test**

Create `tests/Unit/Resources/ProductResourceOctaneTest.php`:

```php
<?php

namespace Tests\Unit\Resources;

use App\Http\Resources\ProductResource;
use Tests\TestCase;

class ProductResourceOctaneTest extends TestCase
{
    /** @test */
    public function it_can_flush_the_wishlist_cache(): void
    {
        // Access the static property via reflection to seed it
        $reflection = new \ReflectionClass(ProductResource::class);
        $property = $reflection->getProperty('wishlistProductIds');
        $property->setAccessible(true);

        // Seed with fake data (simulating a previous request)
        $property->setValue(null, [1 => [10, 20, 30]]);

        // Verify it's seeded
        $this->assertNotEmpty($property->getValue(null));

        // Flush
        ProductResource::flushWishlistCache();

        // Verify it's empty
        $this->assertEmpty($property->getValue(null));
    }
}
```

**Step 2: Run test to verify it fails**

Run:
```bash
php artisan test --compact --filter=ProductResourceOctaneTest
```
Expected: FAIL — `flushWishlistCache` method does not exist.

**Step 3: Add the flushWishlistCache method to ProductResource**

In `app/Http/Resources/ProductResource.php`, add this method after the `checkIsWishlisted` method (before the closing `}`):

```php
/**
 * Flush the static wishlist cache.
 * Called on each request by Octane's RequestReceived listener.
 */
public static function flushWishlistCache(): void
{
    self::$wishlistProductIds = [];
}
```

**Step 4: Run test to verify it passes**

Run:
```bash
php artisan test --compact --filter=ProductResourceOctaneTest
```
Expected: PASS

**Step 5: Commit**

```bash
git add app/Http/Resources/ProductResource.php tests/Unit/Resources/ProductResourceOctaneTest.php
git commit -m "fix: add flushWishlistCache to ProductResource for Octane compatibility"
```

### Task 4: Register Octane RequestReceived Listener

Wire up the flush methods so they run at the start of every Octane request. This prevents the static caches from leaking across requests.

**Files:**
- Modify: `config/octane.php` (add listeners)

**Step 1: Update config/octane.php listeners array**

Find the `'listeners'` key in `config/octane.php`. It will look like:

```php
'listeners' => [
    WorkerStarting::class => [
        //
    ],
    RequestReceived::class => [
        //
    ],
    // ... more events
],
```

Update the `RequestReceived::class` entry to:

```php
RequestReceived::class => [
    \App\Http\Resources\ProductResource::class.'::flushWishlistCache',
    \App\Http\Resources\ServiceResource::class.'::flushWishlistCache',
],
```

> **Important:** Octane listeners expect callables. Static method strings `'Class::method'` work as PHP callables. Alternatively, you can use closures: `fn () => ProductResource::flushWishlistCache()`.

### Task 5: Update OPcache Configuration

Disable JIT (conflicts with Swoole) while keeping OPcache file caching for performance.

**Files:**
- Modify: `docker/php/opcache.ini`

**Step 1: Update opcache.ini**

Replace the full contents of `docker/php/opcache.ini` with:

```ini
; =============================================================================
; OPcache Configuration for SurpriseMoi (Octane + Swoole)
; Note: JIT is disabled because it conflicts with Swoole's coroutine engine
; =============================================================================

[opcache]
opcache.enable = 1
opcache.enable_cli = 1
opcache.memory_consumption = 128
opcache.interned_strings_buffer = 8
opcache.max_accelerated_files = 10000
opcache.validate_timestamps = 0
opcache.revalidate_freq = 0
opcache.save_comments = 1
opcache.fast_shutdown = 1

; JIT disabled - incompatible with Swoole coroutines
opcache.jit = disable
```

> **Key change:** Removed `opcache.jit = 1255` and `opcache.jit_buffer_size = 64M`. Set `opcache.jit = disable`. Without this, Swoole will segfault or behave unpredictably.

### Task 7: Update Nginx Config — FastCGI to Reverse Proxy

Change the in-container Nginx from FastCGI proxying to HTTP reverse proxying to Octane.

**Files:**
- Modify: `docker/nginx/nginx.conf` (add WebSocket upgrade map)
- Modify: `docker/nginx/default.conf` (complete rewrite)

**Step 1: Add the WebSocket upgrade map to nginx.conf**

In `docker/nginx/nginx.conf`, add the `map` directive inside the `http {}` block, before the `include` line. Insert at line 16 (after `http {`):

```nginx
    # WebSocket upgrade mapping (used by Octane and Reverb proxies)
    map $http_upgrade $connection_upgrade {
        default upgrade;
        ''      close;
    }
```

**Step 2: Rewrite default.conf for Octane reverse proxy**

Replace the entire contents of `docker/nginx/default.conf` with:

```nginx
# =============================================================================
# Nginx Server Block for SurpriseMoi Laravel Application (Octane)
# Nginx serves static files and proxies dynamic requests to Octane/Swoole
# =============================================================================

upstream octane {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    listen [::]:80;
    server_name localhost dashboard.surprisemoi.com;
    root /var/www/html/public;
    index index.php;

    charset utf-8;
    client_max_body_size 64M;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # WebSocket proxy for Laravel Reverb
    location /app {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }

    # Handle user-uploaded files (storage) - no immutable cache
    location ^~ /storage/ {
        expires 1d;
        add_header Cache-Control "public";
        access_log off;
        try_files $uri =404;
    }

    # Handle Vite build files (versioned, safe to cache immutably)
    location /build {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
        try_files $uri =404;
    }

    # Handle other static assets with caching
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
        try_files $uri =404;
    }

    # Robots.txt
    location = /robots.txt {
        access_log off;
        log_not_found off;
    }

    # Favicon
    location = /favicon.ico {
        access_log off;
        log_not_found off;
    }

    # All other requests -> try static first, then Octane
    location / {
        try_files $uri $uri/ @octane;
    }

    # Octane reverse proxy
    location @octane {
        set $suffix "";

        if ($uri = /index.php) {
            set $suffix ?$query_string;
        }

        proxy_http_version 1.1;
        proxy_set_header Host $http_host;
        proxy_set_header Scheme $scheme;
        proxy_set_header SERVER_PORT $server_port;
        proxy_set_header REMOTE_ADDR $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;

        proxy_pass http://octane$suffix;
    }

    # Deny access to hidden files (except .well-known)
    location ~ /\.(?!well-known).* {
        deny all;
        access_log off;
        log_not_found off;
    }

    # Deny access to sensitive files
    location ~* ^/(\.env|composer\.(json|lock)|package\.(json|lock)|webpack\.mix\.js)$ {
        deny all;
    }

    # Error pages
    error_page 404 /index.php;
    error_page 500 502 503 504 /50x.html;
    location = /50x.html {
        root /var/lib/nginx/html;
    }
}
```

> **Key changes from original:**
> - Removed `location ~ \.php$ { fastcgi_pass ... }` block entirely
> - Added `upstream octane` block pointing to `127.0.0.1:8000`
> - Changed `location /` from `try_files $uri $uri/ /index.php?$query_string` to `try_files $uri $uri/ @octane`
> - Added `location @octane` block with `proxy_pass http://octane`
> - Reverb WebSocket proxy uses `$connection_upgrade` variable (from map in nginx.conf) instead of hardcoded `"upgrade"`
> - Removed `/up` health check special location (Octane serves it like any route)

**Step 3: Commit**

```bash

git commit -m "feat: update Nginx from FastCGI to Octane reverse proxy"
```

### Task 8: Update Supervisor Config — Replace FPM with Octane

**Files:**
- Modify: `docker/supervisor/supervisord.conf`

**Step 1: Replace the php-fpm program with octane**

In `docker/supervisor/supervisord.conf`, replace the `[program:php-fpm]` block (lines 17-25):

```ini
# REMOVE:
[program:php-fpm]
command=/usr/local/sbin/php-fpm --nodaemonize
autostart=true
```

Replace with:

```ini
[program:octane]
; Laravel Octane with Swoole - keeps the app in memory for high performance
command=php /var/www/html/artisan octane:start --server=swoole --host=127.0.0.1 --port=8000 --max-requests=500
directory=/var/www/html
user=www-data
autostart=true
autorestart=true
priority=5
stopwaitsecs=3600
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0
```

> **Key differences:**
> - Runs `octane:start` instead of `php-fpm --nodaemonize`
> - Binds to `127.0.0.1:8000` (Nginx proxies to this)
> - `--max-requests=500` recycles workers to prevent memory leaks
> - `stopwaitsecs=3600` gives Octane time to finish in-flight requests on shutdown
> - Runs as `www-data` (not root)
