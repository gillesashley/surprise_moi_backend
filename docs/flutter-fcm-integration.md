# Surprise Moi - FCM Push Notification Integration Guide (Flutter)

This document describes how to integrate Firebase Cloud Messaging (FCM) push notifications into the Flutter mobile app. FCM push notifications complement the existing [WebSocket real-time notifications](./flutter-notification-integration.md) by delivering notifications even when the app is in the background or terminated.

## Overview

The Surprise Moi backend sends push notifications via FCM alongside database and WebSocket (Reverb) channels. This means:

- **WebSocket (Reverb)** delivers real-time notifications while the app is in the foreground and connected.
- **FCM** delivers push notifications regardless of app state (foreground, background, or terminated).

Both channels carry the same notification data. Your Flutter app should:

1. Register the device's FCM token with the backend after login.
2. Handle incoming push notifications in all three app states.
3. Remove the FCM token on logout.

2. In `android/build.gradle`, ensure the Google services plugin is applied:

```groovy
// android/build.gradle
buildscript {

    }
```

3. In `android/app/build.gradle`:

```groovy

android {
```

2. In `ios/Runner/AppDelegate.swift`, ensure Firebase is initialized (handled automatically if using FlutterFire CLI).
3. Enable **Push Notifications** and **Background Modes > Remote notifications** in Xcode under Signing & Capabilities.

## Device Token Registration API

After the user logs in and Firebase is initialized, register the device's FCM token with the backend so it knows where to send push notifications.

### Endpoint

```
POST /api/v1/device-tokens
```

**Headers:**
```
Authorization: Bearer <sanctum_token>
Accept: application/json
Content-Type: application/json
```

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | string | Yes | The FCM device token (max 255 chars) |
| `device_name` | string | No | Human-readable device name (max 100 chars) |
| `platform` | string | No | `ios` or `android` |

**Response (201 - New Token):**
```json
{
  "success": true,
  "message": "Device token registered"
}
```

**Response (200 - Existing Token Updated/Reassigned):**
```json
{
  "success": true,
  "message": "Device token updated"
}
```

If the token already exists (e.g., the same device was previously logged in as a different user), the backend reassigns it to the current user.

### Dart Example

```dart
import 'package:dio/dio.dart';
import 'package:firebase_messaging/firebase_messaging.dart';
import 'dart:io' show Platform;

class DeviceTokenService {
  final Dio _dio;

  DeviceTokenService(this._dio);

  /// Register the current device's FCM token with the backend.
  /// Call this after login and whenever the token refreshes.
  Future<void> registerToken() async {
    final token = await FirebaseMessaging.instance.getToken();
    if (token == null) return;

    await _dio.post('/api/v1/device-tokens', data: {
      'token': token,
      'platform': Platform.isIOS ? 'ios' : 'android',
      'device_name': Platform.isIOS ? 'iPhone' : 'Android',
    });
  }

  /// Listen for token refreshes and re-register automatically.
  void listenForTokenRefresh() {
    FirebaseMessaging.instance.onTokenRefresh.listen((newToken) async {
      await _dio.post('/api/v1/device-tokens', data: {
        'token': newToken,
        'platform': Platform.isIOS ? 'ios' : 'android',
      });
    });
  }
}
```

### Validation Errors (422)

```json
{
  "message": "A device token is required.",
  "errors": {
    "token": ["A device token is required."],
    "platform": ["The platform must be ios or android."]
  }
}
```

## Device Token Removal (Logout)

When the user logs out, remove the device token so they stop receiving push notifications on that device.

### Endpoint

```
DELETE /api/v1/device-tokens
```

**Headers:**
```
Authorization: Bearer <sanctum_token>
Accept: application/json
Content-Type: application/json
```

**Request Body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token` | string | Yes | The FCM device token to remove |

**Response (200 - Always):**
```json
{
  "success": true,
  "message": "Device token removed"
}
```

This endpoint is idempotent. If the token does not exist or belongs to another user, it still returns 200.

### Dart Example

```dart
/// Remove the device token before logging out.
Future<void> removeTokenAndLogout() async {
  final token = await FirebaseMessaging.instance.getToken();
  if (token != null) {
    await _dio.delete('/api/v1/device-tokens', data: {
      'token': token,
    });
  }

  // Proceed with Sanctum token revocation / logout
  await _dio.post('/api/v1/logout');
}
```

## Push Notification Payload Shape

When the backend sends an FCM notification, it contains two parts:

### Notification Part (Displayed in System Tray)

| Field | Description | Example |
|-------|-------------|---------|
| `title` | Short notification title | `"Someone liked your video"` |
| `body` | Descriptive message | `"John Doe liked your video"` |

### Data Part (For App Logic / Deep Linking)

| Field | Description | Example |
|-------|-------------|---------|
| `type` | Notification type identifier | `"waw_video_liked"` |
| `action_url` | Relative deep link path | `"/waw/videos/42"` |

### Example Raw FCM Payload

```json
{
  "notification": {
    "title": "Someone liked your video",
    "body": "John Doe liked your video"
  },
  "data": {
    "type": "waw_video_liked",
    "action_url": "/waw/videos/42"
  }
}
```

### Notification Types

These are the same types used across the REST API and WebSocket channels:

| `type` | Example `title` | Example `action_url` |
|--------|-----------------|----------------------|
| `waw_video_liked` | "Someone liked your video" | `/waw/videos/{id}` |
| `new_review` | "New Review" | `/reviews/{id}` |
| `review_replied` | "New Reply to Your Review" | `/reviews/{id}` |
| `order_status_changed` | "Order Shipped" / "Order Confirmed" / etc. | `/orders/{id}` |
| `new_chat_message` | "New Message" | `/conversations/{id}` |
| `vendor_approved` | "Application Approved" | `/dashboard/vendor-applications/{id}` |
| `vendor_rejected` | "Application Rejected" | `/dashboard/vendor-applications/{id}` |

## Handling Notifications in Flutter

Push notifications arrive in three different app states. Each requires specific handling.

### 1. Foreground (App is Open)

When the app is in the foreground, FCM delivers the message silently (no system tray notification by default). Use `FirebaseMessaging.onMessage` to handle it:

```dart
import 'package:firebase_messaging/firebase_messaging.dart';
import 'package:flutter_local_notifications/flutter_local_notifications.dart';

final FlutterLocalNotificationsPlugin localNotifications =
    FlutterLocalNotificationsPlugin();

void setupForegroundHandler() {
  FirebaseMessaging.onMessage.listen((RemoteMessage message) {
    final notification = message.notification;
    final data = message.data;

    // Show a local notification so the user sees it
    if (notification != null) {
      localNotifications.show(
        notification.hashCode,
        notification.title,
        notification.body,
        const NotificationDetails(
          android: AndroidNotificationDetails(
            'surprise_moi_notifications',
            'Notifications',
            channelDescription: 'Surprise Moi push notifications',
            importance: Importance.high,
            priority: Priority.high,
            icon: '@mipmap/ic_launcher',
          ),
          iOS: DarwinNotificationDetails(
            presentAlert: true,
            presentBadge: true,
            presentSound: true,
          ),
        ),
        payload: data['action_url'],
      );
    }

    // Update local notification state (badge count, list, etc.)
    onNotificationReceived(data);
  });
}
```

### 2. Background (App is in Background)

When the app is in the background, FCM automatically displays the notification in the system tray using the `notification` payload. When the user taps the notification, `FirebaseMessaging.onMessageOpenedApp` fires:

```dart
void setupBackgroundTapHandler() {
  FirebaseMessaging.onMessageOpenedApp.listen((RemoteMessage message) {
    final data = message.data;
    handleDeepLink(
      type: data['type'],
      actionUrl: data['action_url'],
    );
  });
}
```

For background data processing (without user interaction), register a top-level handler:

```dart
// This MUST be a top-level function (not a class method)
@pragma('vm:entry-point')
Future<void> firebaseMessagingBackgroundHandler(RemoteMessage message) async {
  await Firebase.initializeApp();
  // Perform lightweight work only (e.g., update badge count in local storage)
  // Do NOT access UI or navigation here
}

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);

  FirebaseMessaging.onBackgroundMessage(firebaseMessagingBackgroundHandler);

  runApp(const MyApp());
}
```

### 3. Terminated (App was Killed)

When the user taps a notification that launched the app from a terminated state, retrieve the initial message:

```dart
Future<void> handleTerminatedLaunch() async {
  final initialMessage = await FirebaseMessaging.instance.getInitialMessage();

  if (initialMessage != null) {
    final data = initialMessage.data;
    handleDeepLink(
      type: data['type'],
      actionUrl: data['action_url'],
    );
  }
}
```

Call this once during app initialization, after `Firebase.initializeApp()`.

### Requesting Permissions (iOS)

On iOS, you must request notification permissions before notifications can be delivered:

```dart
Future<void> requestNotificationPermission() async {
  final settings = await FirebaseMessaging.instance.requestPermission(
    alert: true,
    badge: true,
    sound: true,
    provisional: false,
  );

  if (settings.authorizationStatus == AuthorizationStatus.authorized) {
    // User granted permission
  } else if (settings.authorizationStatus == AuthorizationStatus.provisional) {
    // User granted provisional permission (quiet notifications)
  } else {
    // User declined — show in-app prompt explaining why notifications matter
  }
}
```

On Android 13+ (API 33), runtime notification permission is also required. The `firebase_messaging` package handles this automatically when you call `requestPermission()`.

## Deep Linking from Push Notifications

The `action_url` field in the data payload provides a relative path for navigating to the relevant screen. Parse this path and map it to your Flutter routes.

### Route Mapping

| `action_url` Pattern | Flutter Screen |
|---|---|
| `/waw/videos/{id}` | Waw video detail |
| `/reviews/{id}` | Review detail |
| `/orders/{id}` | Order detail |
| `/conversations/{id}` | Chat conversation |
| `/dashboard/vendor-applications/{id}` | Vendor application status |

### Deep Link Handler

```dart
void handleDeepLink({String? type, String? actionUrl}) {
  if (actionUrl == null || actionUrl.isEmpty) return;

  final uri = Uri.parse(actionUrl);
  final segments = uri.pathSegments;

  // Parse the action_url and navigate accordingly
  if (segments.length >= 3 && segments[0] == 'waw' && segments[1] == 'videos') {
    final videoId = int.tryParse(segments[2]);
    if (videoId != null) {
      navigatorKey.currentState?.pushNamed('/waw/video-detail', arguments: videoId);
      return;
    }
  }

  if (segments.length >= 2 && segments[0] == 'reviews') {
    final reviewId = int.tryParse(segments[1]);
    if (reviewId != null) {
      navigatorKey.currentState?.pushNamed('/review-detail', arguments: reviewId);
      return;
    }
  }

  if (segments.length >= 2 && segments[0] == 'orders') {
    final orderId = int.tryParse(segments[1]);
    if (orderId != null) {
      navigatorKey.currentState?.pushNamed('/order-detail', arguments: orderId);
      return;
    }
  }

  if (segments.length >= 2 && segments[0] == 'conversations') {
    final conversationId = int.tryParse(segments[1]);
    if (conversationId != null) {
      navigatorKey.currentState?.pushNamed('/chat', arguments: conversationId);
      return;
    }
  }

  if (segments.length >= 3 &&
      segments[0] == 'dashboard' &&
      segments[1] == 'vendor-applications') {
    final applicationId = int.tryParse(segments[2]);
    if (applicationId != null) {
      navigatorKey.currentState?.pushNamed('/vendor-application', arguments: applicationId);
      return;
    }
  }
}
```

## Complete Implementation Example

Below is a full-featured `PushNotificationService` class that ties together all the concepts above.

### push_notification_service.dart

```dart
import 'dart:convert';
import 'dart:io' show Platform;

import 'package:dio/dio.dart';
import 'package:firebase_core/firebase_core.dart';
import 'package:firebase_messaging/firebase_messaging.dart';
import 'package:flutter/material.dart';
import 'package:flutter_local_notifications/flutter_local_notifications.dart';

/// Top-level background message handler.
/// Must be a top-level function, not a class method.
@pragma('vm:entry-point')
Future<void> _firebaseMessagingBackgroundHandler(RemoteMessage message) async {
  await Firebase.initializeApp();
  // Lightweight background work only (e.g., update local badge count)
}

class PushNotificationService {
  final Dio _dio;
  final GlobalKey<NavigatorState> navigatorKey;
  final void Function(Map<String, dynamic> data)? onNotificationReceived;

  late final FlutterLocalNotificationsPlugin _localNotifications;
  late final FirebaseMessaging _messaging;

  PushNotificationService({
    required this.navigatorKey,
    required Dio dio,
    this.onNotificationReceived,
  }) : _dio = dio;

  // ---------------------------------------------------------------------------
  // Initialization
  // ---------------------------------------------------------------------------

  /// Call this once during app startup, after Firebase.initializeApp().
  Future<void> initialize() async {
    _messaging = FirebaseMessaging.instance;
    _localNotifications = FlutterLocalNotificationsPlugin();

    // Register the background handler
    FirebaseMessaging.onBackgroundMessage(_firebaseMessagingBackgroundHandler);

    // Initialize local notifications for foreground display
    await _initializeLocalNotifications();

    // Request permission (required on iOS, Android 13+)
    await _requestPermission();

    // Set up listeners for all three app states
    _setupForegroundListener();
    _setupBackgroundTapListener();
    await _handleTerminatedLaunch();
  }

  // ---------------------------------------------------------------------------
  // Permission
  // ---------------------------------------------------------------------------

  Future<void> _requestPermission() async {
    final settings = await _messaging.requestPermission(
      alert: true,
      badge: true,
      sound: true,
      provisional: false,
    );

    debugPrint(
      'FCM permission status: ${settings.authorizationStatus}',
    );
  }

  // ---------------------------------------------------------------------------
  // Local Notifications Setup
  // ---------------------------------------------------------------------------

  Future<void> _initializeLocalNotifications() async {
    const androidSettings = AndroidInitializationSettings('@mipmap/ic_launcher');
    const iosSettings = DarwinInitializationSettings(
      requestAlertPermission: false, // Already handled by FCM
      requestBadgePermission: false,
      requestSoundPermission: false,
    );

    await _localNotifications.initialize(
      const InitializationSettings(
        android: androidSettings,
        iOS: iosSettings,
      ),
      onDidReceiveNotificationResponse: (NotificationResponse response) {
        // User tapped a local notification shown while app was in foreground
        if (response.payload != null && response.payload!.isNotEmpty) {
          _handleDeepLink(actionUrl: response.payload);
        }
      },
    );

    // Create the Android notification channel
    const channel = AndroidNotificationChannel(
      'surprise_moi_notifications',
      'Notifications',
      description: 'Surprise Moi push notifications',
      importance: Importance.high,
    );

    await _localNotifications
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>()
        ?.createNotificationChannel(channel);
  }

  // ---------------------------------------------------------------------------
  // Device Token Registration
  // ---------------------------------------------------------------------------

  /// Register the current device's FCM token with the backend.
  /// Call after login and whenever the token refreshes.
  Future<void> registerDeviceToken() async {
    final token = await _messaging.getToken();
    if (token == null) return;

    await _dio.post('/api/v1/device-tokens', data: {
      'token': token,
      'platform': Platform.isIOS ? 'ios' : 'android',
      'device_name': Platform.isIOS ? 'iPhone' : 'Android',
    });

    // Listen for token refreshes and re-register automatically
    _messaging.onTokenRefresh.listen((newToken) async {
      await _dio.post('/api/v1/device-tokens', data: {
        'token': newToken,
        'platform': Platform.isIOS ? 'ios' : 'android',
      });
    });
  }

  /// Remove the device token before logout.
  Future<void> removeDeviceToken() async {
    final token = await _messaging.getToken();
    if (token == null) return;

    try {
      await _dio.delete('/api/v1/device-tokens', data: {
        'token': token,
      });
    } catch (_) {
      // Best-effort removal — don't block logout on failure
    }
  }

  // ---------------------------------------------------------------------------
  // Foreground Notifications
  // ---------------------------------------------------------------------------

  void _setupForegroundListener() {
    FirebaseMessaging.onMessage.listen((RemoteMessage message) {
      final notification = message.notification;
      final data = message.data;

      // Show a local notification so the user sees it in the foreground
      if (notification != null) {
        _localNotifications.show(
          notification.hashCode,
          notification.title,
          notification.body,
          const NotificationDetails(
            android: AndroidNotificationDetails(
              'surprise_moi_notifications',
              'Notifications',
              channelDescription: 'Surprise Moi push notifications',
              importance: Importance.high,
              priority: Priority.high,
              icon: '@mipmap/ic_launcher',
            ),
            iOS: DarwinNotificationDetails(
              presentAlert: true,
              presentBadge: true,
              presentSound: true,
            ),
          ),
          payload: data['action_url'],
        );
      }

      // Notify the app state (update badge count, prepend to list, etc.)
      onNotificationReceived?.call(data);
    });
  }

  // ---------------------------------------------------------------------------
  // Background Tap Handler
  // ---------------------------------------------------------------------------

  void _setupBackgroundTapListener() {
    FirebaseMessaging.onMessageOpenedApp.listen((RemoteMessage message) {
      _handleDeepLink(
        type: message.data['type'],
        actionUrl: message.data['action_url'],
      );
    });
  }

  // ---------------------------------------------------------------------------
  // Terminated Launch Handler
  // ---------------------------------------------------------------------------

  Future<void> _handleTerminatedLaunch() async {
    final initialMessage = await _messaging.getInitialMessage();

    if (initialMessage != null) {
      // Delay slightly to ensure the navigator is ready
      Future.delayed(const Duration(milliseconds: 500), () {
        _handleDeepLink(
          type: initialMessage.data['type'],
          actionUrl: initialMessage.data['action_url'],
        );
      });
    }
  }

  // ---------------------------------------------------------------------------
  // Deep Linking
  // ---------------------------------------------------------------------------

  void _handleDeepLink({String? type, String? actionUrl}) {
    if (actionUrl == null || actionUrl.isEmpty) return;

    final uri = Uri.parse(actionUrl);
    final segments = uri.pathSegments;

    if (segments.length >= 3 && segments[0] == 'waw' && segments[1] == 'videos') {
      final videoId = int.tryParse(segments[2]);
      if (videoId != null) {
        navigatorKey.currentState?.pushNamed('/waw/video-detail', arguments: videoId);
        return;
      }
    }

    if (segments.length >= 2 && segments[0] == 'reviews') {
      final reviewId = int.tryParse(segments[1]);
      if (reviewId != null) {
        navigatorKey.currentState?.pushNamed('/review-detail', arguments: reviewId);
        return;
      }
    }

    if (segments.length >= 2 && segments[0] == 'orders') {
      final orderId = int.tryParse(segments[1]);
      if (orderId != null) {
        navigatorKey.currentState?.pushNamed('/order-detail', arguments: orderId);
        return;
      }
    }

    if (segments.length >= 2 && segments[0] == 'conversations') {
      final conversationId = int.tryParse(segments[1]);
      if (conversationId != null) {
        navigatorKey.currentState?.pushNamed('/chat', arguments: conversationId);
        return;
      }
    }

    if (segments.length >= 3 &&
        segments[0] == 'dashboard' &&
        segments[1] == 'vendor-applications') {
      final applicationId = int.tryParse(segments[2]);
      if (applicationId != null) {
        navigatorKey.currentState?.pushNamed(
          '/vendor-application',
          arguments: applicationId,
        );
        return;
      }
    }
  }
}
```

## Important Notes

1. **Token refresh**: FCM tokens can change at any time. Always listen to `onTokenRefresh` and re-register with the backend.
2. **Multiple devices**: A user can have multiple device tokens (one per device). The backend stores all of them and sends push notifications to every registered device.
3. **Token reassignment**: If a device token already exists for a different user (e.g., shared device), the backend automatically reassigns it to the current user on registration.
4. **Idempotent removal**: The `DELETE /api/v1/device-tokens` endpoint always returns 200, even if the token was already removed or does not exist. This makes logout flows safe to retry.
5. **Background handler isolation**: The `_firebaseMessagingBackgroundHandler` runs in a separate isolate. It cannot access UI, navigation, or in-memory app state. Keep it lightweight.
6. **iOS APNs**: FCM on iOS relies on Apple Push Notification service (APNs). Ensure your APNs authentication key is uploaded in Firebase Console under Project Settings > Cloud Messaging.
7. **`action_url` is a relative path**, not a full URL. Map these paths to your app's route/screen system as shown in the deep linking section.
8. **Consistency**: The `type` and `action_url` fields in the FCM data payload match exactly what the REST API and WebSocket channels return. You can reuse the same deep link handler across all notification sources.
